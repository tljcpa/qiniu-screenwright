# Screenwright Demo 视频 分镜 + 旁白脚本

目标时长约 90 秒。站点 https://script.qiniu.zdwktlj.top 。桌面 1440x900 录制（勿用窄屏）。
演示走"加载样例"缓存路径（秒出，不让镜头干等）。全程不点没必要的控件，按下面节奏走。

| # | 画面动作 | 旁白(中文) | 时长 |
|---|---------|-----------|------|
| 1 | 停在首页，露出产品名 Screenwright 编剧工作台 | "Screenwright，一个把小说变成可拍剧本的智能编剧工作台。" | 6s |
| 2 | 点"加载样例"，剧本秒出，镜头扫过三栏 | "上传小说，它先理解整个故事世界——人物、地点、时间线，再逐场生成结构化剧本。左边原文，中间剧本，右边质量看板。" | 12s |
| 3 | 点剧本中一行对白，左侧原文对应句高亮；再点左侧原文一段，右侧剧本高亮 | "每一行剧本都能点回它来自的原文，双向可查。作者一眼就能审计、打磨——这是行级双向溯源。" | 14s |
| 4 | 鼠标悬停/指向"内心戏 → 动作"标签 | "小说里不可拍的内心独白，被自动外化成动作或潜台词，并透明标记，可一键回溯。" | 10s |
| 5 | 顶栏媒介从"电影"切到"短剧"，剧本重生成、结构变化 | "同一部小说，一键切换电影、剧集、短剧。切到短剧，节奏立刻变得更快、更强冲突——契合当下的短剧市场。" | 14s |
| 6 | 某场点"重生成"，填指令"更紧张"，确认，该场刷新（其他场不动） | "任何一场都能下指令单独重生成，而且编辑安全——只改这一场，其余原样不动。" | 12s |
| 7 | 指向质量看板（溯源覆盖率/外化数/冲突0），再依次点导出 YAML / Fountain / PDF | "质量看板把溯源覆盖率、外化数量、连贯性冲突量化呈现；剧本可一键导出 YAML、Fountain、PDF，既能机读也能手改。" | 14s |
| 8 | 回到全局视图定格 | "从小说到可拍剧本，Screenwright 降低改编门槛。在线即可试用。" | 8s |

注意事项（来自拟人 E2E）：
- 务必桌面端录制，窄屏布局会堆叠（虽已修复可用，但桌面更佳）。
- 走"加载样例"避免真实生成的等待；如要展示进度条，可单独补录一次粘贴生成的进度片段。
- 重生成、媒介切换均已实测可用，可放心入镜。
